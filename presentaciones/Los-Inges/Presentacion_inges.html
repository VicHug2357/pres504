CLASES DE RELACIÓN

Las clases de relación se refieren a la manera en que las entidades se conectan entre sí. Estas relaciones definen cómo las diferentes tablas de la base de datos se relacionan entre sí y cómo los datos en una tabla están vinculados a los datos en otra.

Las clases de relación comunes incluyen:

1. Uno a Uno (1:1)-Una instancia de una entidad en una tabla se asocia con exactamente una instancia de otra entidad en otra tabla.
2. Uno a Muchos (1: N) -Una instancia de una entidad en una tabla se asocia con cero o más instancias de otra entidad en otra tabla.
3. Muchos a Uno (N:1) -Muchas instancias de una entidad en una tabla se asocian con una instancia de otra entidad en otra tabla.
4. Muchos a Muchos (N:M) -Muchas instancias de una entidad en una tabla se asocian con muchas instancias de otra entidad en otra tabla.

DATOS PERSISTENTES

Son aquellos datos que se mantienen almacenados en la base de datos incluso después de que la aplicación que los creó haya finalizado. Esto significa que los datos persisten en el tiempo y están disponibles para ser recuperados y utilizados en cualquier momento.
Ejemplo: Una base de datos de usuarios en una web. Aunque los usuarios puedas cerrar sesión o incluso abandonar el sitio, sus datos de registro permanecerán almacenados en la base de datos hasta que sean eliminados o modificados.

DATOS TEMPORALES

Son aquellos datos que son validos durante un periodo específico de tiempo. Estos datos pueden ser relevantes solo para una tarea o evento específico y pueden eliminarse o actualizarse después de que espire su relevancia temporal.
Ejemplo: Registro de transacciones financieras en una aplicación bancaria. A media de que pasa el tiempo, algunas transacciones pueden archivarse o eliminarse de la base de datos, mientras que otras pueden mantenerse para futuras referencias o auditorias.
