¿Qué es el esquema de la relación?

Es la estructura logica que describe la organizacion de los datos en una base relacional.

Define como se organiza la informacion en tablas, especificando los campos o atributos que cada tabla contiene y su relacion entre estas.

¿Qué lo caracteriza?

Su nombre, un conjunto de atributos y una coleccion de dominios.

Ademas, cabe recalcar que se puede expresar utilizando notacion matematica, transmitiendo el proposito y sus restricciones

Elementos de esquema de la relacion

  1. Nombre de la tabla
  2. Atributos o campos
  3. Tipos de datos
  4. Restricciones de integridad

Tipos de datos

Aunque hay bastantes tipos de datos, entre los mas usados podemos encontrar:

  1. Integer o INT: Numeros enteros
  2. Decimal/Float o FLOAT/NUMERIC/DECIMAL: Numeros con decimales
  3. String/Char o CHAR/VARCHAR: Cadenas de caracteres
  4. DATE o TIME o DATETIME: Fechas, horas o ambas
  5. BOOLEAN: True o False
  6. BINARY: Archivos binarios como imagenes o archivos
  7. ENUM: Varios valores predefinidos por uno mismo
  8. JSON: Datos en forma JSON para apps web o moviles
  9. XML: Formato de intercambio de datos para aplicaciones web u otros

Restricciones

Son reglas que aplican a los datos alamcenados una precision, coherencia e integridad

  1. PK o Primary Key: Identifica de forma unica cada fila en una tabla
  2. FK o Foreign Key: Llave que viene de otra tabla como PK
  3. NN o NOTNULL: Asegura que un campo no pueda contener valores nulos
  4. U o UNIQUE: Asegura que todos los valores en una columna sean unicos
  5. CH o CHECK: Permite especificar una condicion que debe cupmlirse para los valores en una columna
  6. DC o DEFAULT CONSTRAINT: Establece un valor predeterminado para una columna cuando no se proporciona un valor durante la inserción de datos
  7. EIC o ENTITY INTREGITY CONSTRAINT: Se refiere a la combinación de claves primarias y restricciones NOT NULL para garantizar que cada fila en una tabla sea única y que ningún valor clave sea nulo.

Especificidad Muchos a Muchos

Cuando se trata de una relacion muchos a muchos, es mas complicado a diferencia del resto, pues para este caso se debe agregar una tabla en la cual se agregan FK de ambas para determinar su relacion.

Ejemplo Esquema:

Nombre de la relación: Empleados

Atributos (cabecera de la relación):

  1. ID_empleado (clave primaria)
  2. Nombre
  3. Apellido
  4. Departamento
  5. Cargo
  6. Salario

Ejemplo Esquema:

Restricciones de integridad:

  • Clave primaria: ID_empleado
  • Restricción de unicidad: No puede haber dos empleados con el mismo ID_empleado.
  • Restricciones de tipo de datos:
    • El ID_empleado puede ser un número entero.
    • El nombre y apellido son cadenas de caracteres.
    • El salario puede ser un número decimal.
