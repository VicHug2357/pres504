¿Que son las restricciones?

Son reglas y condiciones que se aplican a los datos almacenados para garantizar su integridad, consistencia y validez

Restricciones inherentes al modelo relacional

Son reglas predefinidas que se aplican a las tablas de una base de datos relacional para garantizar la integridad y consistencia de los datos. Estas restricciones ayudan a mantener la calidad de la información almacenada y a prevenir inconsistencias o errores en la base de datos.

Tipos y Ejemplos

  1. Restricción de Integridad Referencial:

    Esta restricción asegura que las relaciones entre tablas sean coherentes. Por ejemplo, en una tabla de "Pedidos", la columna de "ID de Cliente" solo puede contener valores que existan en la tabla de "Clientes". Esto evita que se hagan pedidos para clientes que no están registrados en la base de datos.

2.Restricción de Unicidad:

Esta restricción asegura que los valores en una columna específica de una tabla sean únicos. Por ejemplo, En una tabla de "Productos", la columna de "Número de Serie" no puede tener el mismo número en más de un producto. Esto asegura que cada producto tenga un número de serie único.

3.Restricción de Clave Primaria:

Esta restricción identifica de manera única cada fila en una tabla y garantiza que los valores de la clave primaria no puedan ser nulos ni duplicados. Por ejemplo, En una tabla de "Empleados", la columna de "ID de Empleado" actúa como clave primaria, asegurando que cada empleado tenga un identificador único.

4.Restricción de Clave Externa:

Esta restricción garantiza la consistencia en las relaciones entre tablas al asegurar que los valores en una columna que hace referencia a una clave primaria en otra tabla sean válidos. Por ejemplo, En una tabla de "Detalle de Pedido", la columna de "ID de Producto" debe coincidir con un producto existente en la tabla de "Productos". Esto garantiza que solo se puedan hacer pedidos de productos que estén en el inventario.

Conclusion

Las restricciones inherentes del modelo relacional son reglas fundamentales que se aplican a las tablas de una base de datos relacional para mantener la integridad y coherencia de los datos almacenados.

RESTRICCIONES SEMANTICAS

Las restricciones semanticas se refieren a reglas que se aplican al significado de los datos almacenados en la base de datos.
Estas restricciones garantizan la integridad de los datos y ayudan a mantener la consistencia y la calidad de la infomacion almacenada.
Estas restricciones se dividen en:

1. RESTRICCION DE CLAVE PRIMARIA (PRIMARY KEY).

Permite declarar un atributo o conjunto de atributos como la clave primaria de una relación.

2. RESTRICCION DE UNIDAD (UNIQUE).

Permite que una clave alternativa o secundaria pueda tomar valores únicos para las tuplas de una relación (como si de una clave primaria se tratara).

3. RESTRICCION DE OBLIGATORIEDAD (NOT NULL).

Permite declarar si uno o varios atributos de una relación, esta debe tomar siempre un valor.

4. RESTRICCION DE INTEGRIDAD REFERENCIAL O DE CLAVE FORANEA (FOREING KEY).

Se utiliza para que mediante claves foráneas podamos enlazar relaciones de una base de datos. La integridad referencial nos indica que si una relación tiene una clave foránea que referencia a otra relación, cada valor de la clave foránea o ajena tiene que ser igual a un valor de la clave principal de la relación a la que referencia, o bien, ser completamente nulo.

5. RESTRICCION DE VALOR POR DEFECTO (DEFAULT).

Permite que cuando se inserte una tupla o registro en una tabla, en aquellos atributos para los cuales no se indique un valor exacto se les asigne un valor por defecto.

6. RESTRICCION DE VERIFICACION O CHEQUEO (CHECK).

En algunos casos puede ocurrir que sea necesario especificar una condición que deben cumplir los valores de determinados atributos de una relación de la BD, aparte de las restricciones vistas anteriormente.

7. ASERCIONES (ASSERTION).

Esta restricción generaliza a la anterior, lo forman las aserciones en las que la condición se establece sobre elementos de distintas relaciones (por ello debe tener un nombre que la identifique).

8. DISPARADORES (TRIGGERS).

Los disparadores pueden interpretarse como reglas del tipo evento-condición-acción (ECA) que pueden interpretarse como reglas que especifican que cuando se produce un evento, si se cumple una condición, entonces se realiza una determinada acción.

EJEMPLO

Se tiene una tabla llamada "PRODUCTOS" con las siguientes columnas:
-ID_producto
-Nombre
-Precio
-Stock
Para asegurar que el precio de un producto sea mayor que cero, se puede aplicar la sexta restriccion (CHECK constraint) en la columna precio.
Esta restriccion garantiza que solo se puedan insertar o actualizar valores positivos en la columna "precio" de la tabla "PRODUCTOS", lo que asegura la integridad de los datos.

Operación restringida (restrict)

Es el comportamiento por defecto, que impide realizar modificaciones que atentan contra la integridad referencial. El comportamiento por defecto de una restricción de clave externa es impedir un cambio en la base de datos como consecuencia de una sentencia DELETE o UPDATE, si esta trajese como consecuencia un fallo de la integridad referencial.
