Restricciones de Rechazo en Bases de Datos

Verificación

  • Definición: Las restricciones de verificación se utilizan para garantizar que los datos en una base de datos cumplan ciertas condiciones.
  • Ejemplo: En una tabla de empleados, una restricción de verificación podría ser que la edad de un empleado debe ser mayor de 18 años.

Aserción

  • Definición: Las restricciones de aserción son condiciones lógicas que se aplican a nivel de fila o tabla completa para garantizar la integridad de los datos.
  • Ejemplo: En una tabla de estudiantes, una restricción de aserción podría ser que la suma de las calificaciones de todas las asignaturas no debe exceder 100.

Ejemplos

Restricción de Verificación

CREATE TABLE Empleados (
    ID INT PRIMARY KEY,
    Nombre VARCHAR(50),
    Edad INT CHECK (Edad > 18),
    Departamento VARCHAR(50)
);

Restricción de Aserción

CREATE TABLE Estudiantes (
    ID INT PRIMARY KEY,
    Nombre VARCHAR(50),
    Matematicas INT,
    Ciencias INT,
    Ingles INT,
    CONSTRAINT Suma_Calificaciones CHECK (Matematicas + Ciencias + Ingles <= 100)
);

Conclusiones

  • Las restricciones de rechazo son fundamentales para mantener la integridad de los datos en una base de datos.
  • La verificación y la aserción son dos tipos comunes de restricciones de rechazo.
  • Es importante diseñar cuidadosamente estas restricciones para garantizar la consistencia y la precisión de los datos.
