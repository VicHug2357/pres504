DOMINIO "D"

Un dominio D se refiere al conjunto de valores que pueden tomar los atributos de una relación. Es decir, especifica el conjunto de valores que son válidos para un atributo en particular.

EJEMPLO 1

Si tienes una relación de empleados, podrías tener un atributo "edad" con un dominio D que incluye valores enteros no negativos. Del mismo modo, podrías tener un atributo "nombre" con un dominio que incluye cadenas de caracteres de longitud limitada.

EJEMPLO 2

Puede que necesite enmascarar todas las instancias de números de la seguridad social con la regla de enmascaramiento .Se puede crear un dominio de datos que describa los datos de la seguridad social que aparecen en distintas columnas. Una base de datos podría tener un número de la seguridad social en una columna llamada SSN. La base de datos también tiene una columna llamada SOCIAL_SECURITY en otra tabla. Un número de la seguridad social también podría aparecer en una columna COMMENTS.

Configure dominios de datos para agrupar columnas de origen de datos para el enmascaramiento de datos.

regresar

Ejemplo de aplicación

Supongamos que estás creando una base de datos para una tienda en línea que vende productos electrónicos. En esta base de datos, tendrás una tabla llamada «Productos» que incluye un campo llamado «Categoría«. El dominio de este campo podría ser un conjunto de valores, como «Teléfonos móviles«, «Tabletas«, «Accesorios«, etc. Cada producto en la tabla tendrá un valor en el campo «Categoría» que se ajuste a este dominio.

TIPOS DE DOMINIOS

Dominios por Intención (o definición):

En este enfoque, el dominio se define mediante una descripción o una regla que determina qué valores son válidos para un atributo. Los valores pueden no estar explícitamente enumerados, pero se describen mediante una fórmula o una regla.
Ejemplo:
Dominio de Edad: Todos los números enteros mayores o iguales que cero y menores que 150 son válidos para representar edades.

Dominios por Extensión:

Aquí, el dominio se especifica enumerando todos los posibles valores válidos para el atributo. Es una lista explícita de valores que pueden tomar los atributos.

Ejemplo:

Dominio de Género: {Masculino, Femenino, No binario, Otro}
